# FlowDoc API Reference ‚Äî ERP Integration

Base URL: `https://app.flow-doc.ai`

---

## Authentication

All endpoints require authentication via API key.

```
Authorization: Bearer fd_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

API keys are generated by a FlowDoc admin. Each key is tied to a company and shown only once at creation time.

### Error Responses

| Status | Body | Meaning |
|--------|------|---------|
| 401 | `{ "error": "No autorizado" }` | Missing or invalid API key |
| 403 | `{ "error": "Solo administradores" }` | Admin-only endpoint |
| 400 | `{ "error": "..." }` | Bad request (details in message) |
| 500 | `{ "error": "..." }` | Server error |

Note: API error messages are in Spanish (the platform serves Dominican Republic businesses).

**Response:** Array of queue items:

```json
    "id": "uuid",
    "company_id": "uuid",
    "invoice_id": "uuid",
    "status": "queue",
    "batch_id": null,
    "error_message": null,

    "proveedor": "Empresa XYZ"
  }
]
```

Queue statuses: `queue` ‚Üí `syncing` ‚Üí `done` | `error` | `dup`

#### `POST /api/erp/sync/claim`

**This is the recommended way to pick up invoices for processing.** Atomically claims queued items (marks them as `syncing`) and returns them with full invoice + vendor data. Uses `FOR UPDATE SKIP LOCKED` to prevent duplicate processing across concurrent middleware instances.

```json
{
  "limit": 10
}
```

#### `POST /api/invoices/:id/approve`

Approve an invoice. Adds it to ERP sync queue if `auto_queue_erp` is enabled.

**Request Body:**
```json

```

`tasaCambio` is required for foreign currency invoices (moneda !== "DOP"). Returns `422` if missing.

### Documents

#### `GET /api/documents/:invoiceId`

Get the invoice document. **Supported formats: PDF, JPEG, PNG, WebP.** Check the `mimeType` field or `Content-Type` header to know which format you're receiving.

**Query Parameters:**
| Param | Type | Description |
|-------|------|-------------|
| `download` | "1" | Download raw file bytes with `Content-Disposition: attachment` header |
| `proxy` | "1" | Stream file through server instead of signed URL |

**Default response** (no params): JSON with signed URL
```json
{
  "url": "https://...supabase.co/storage/v1/object/sign/...",
  "mimeType": "application/pdf",
  "filename": "factura.pdf"
}
```

With `?download=1`: Raw file bytes. Use the `Content-Type` header to determine format:
- `application/pdf` ‚Äî PDF document
- `image/jpeg` ‚Äî JPEG photo
- `image/png` ‚Äî PNG image
- `image/webp` ‚Äî WebP image

### Master Data Sync (SAP ‚Üí FlowDoc)

Use these endpoints to sync master data from SAP so that FlowDoc users can select from dropdowns instead of typing codes manually.

#### `POST /api/erp/master-data`

Push master data from SAP to FlowDoc. Replaces all existing active records of the given type.

**Request Body:**
```json
{
  "type": "gl_accounts",
  "items": [
    { "code": "61010101", "name": "Gastos generales" },
    { "code": "61010201", "name": "Servicios profesionales" }
  ]
}
```

Supported types:
| Type | Description | Extra fields |
|------|-------------|--------------|
| `gl_accounts` | Chart of Accounts (GL) | ‚Äî |
| `cost_centers` | Cost Centers | ‚Äî |
| `projects` | Projects | ‚Äî |
| `tax_codes` | Tax Codes | `rate` (numeric, e.g., 18.00) |

Tax codes example:
```json
{
  "type": "tax_codes",
  "items": [
    { "code": "ITBIS", "name": "ITBIS 18%", "rate": 18.00 },
    { "code": "EXENTO", "name": "Exento", "rate": 0 }
  ]
}
```

**Response:**
```json
{
  "type": "gl_accounts",
  "synced": 25,
  "message": "25 registros sincronizados"
}
```

#### `GET /api/erp/master-data?type=gl_accounts`

Read synced master data. Used by the FlowDoc UI for dropdowns.

**Query Parameters:**
| Param | Type | Description |
|-------|------|-------------|
| `type` | string | Required. One of: `gl_accounts`, `cost_centers`, `projects`, `tax_codes` |

**Response:**
```json
{
  "type": "gl_accounts",
  "items": [
    { "code": "61010101", "name": "Gastos generales" },
    { "code": "61010201", "name": "Servicios profesionales" }
  ]
}
```

## Recommended Middleware Flow

This is the typical polling loop your middleware should implement:

```
1. POST /api/erp/sync/claim { limit: 10 }
   ‚Üí Atomically claims up to 10 queued invoices
   ‚Üí Returns batchId + items with full invoice/vendor data

2. For each claimed item:
   a. GET /api/documents/{invoice_id}?download=1
      ‚Üí Download the PDF/image (check Content-Type header)

   b. Connect to SAP B1 Service Layer:
      - Find or create vendor (BusinessPartners)
      - Check for duplicate NCF (PurchaseInvoices filter)
      - Create Purchase Invoice with line items + attachment

3. POST /api/erp/webhook
   {
     "batchId": "...",
     "results": [
       { "invoiceId": "...", "success": true, "erpDocId": "12345" },
       { "invoiceId": "...", "success": false, "errorCode": "...", "errorMessage": "..." }
     ]
   }

4. Periodically sync master data:
   - GET chart of accounts from SAP ‚Üí POST /api/erp/master-data { type: "gl_accounts", items: [...] }
   - GET cost centers from SAP ‚Üí POST /api/erp/master-data { type: "cost_centers", items: [...] }
   - GET tax codes from SAP ‚Üí POST /api/erp/master-data { type: "tax_codes", items: [...] }
   - GET projects from SAP ‚Üí POST /api/erp/master-data { type: "projects", items: [...] }
```

## Data Model (Key Tables)

### invoices
| Column | Type | Description |
|--------|------|-------------|
| id | UUID | Primary key |
| ncf | VARCHAR | NCF (e-NCF E31, paper B01, etc.) or invoice number for international |
| ncf_type | ENUM | E31, B01, E34, B02, international, etc. |
| fecha_emision | DATE | Invoice date |
| subtotal | NUMERIC(15,2) | Subtotal before taxes |
| descuento | NUMERIC(15,2) | Discount amount |
| impuesto | NUMERIC(15,2) | ITBIS (18% tax) |
| propina_legal | NUMERIC(15,2) | Legal tip (10%) |
| propina_adicional | NUMERIC(15,2) | Additional tip |
| isc | NUMERIC(15,2) | Selective consumption tax |
| otros_cargos | NUMERIC(15,2) | Other charges |
| total | NUMERIC(15,2) | Total = subtotal - descuento + impuesto + propinas + isc + otros |
| moneda | VARCHAR | DOP or USD |
| tasa_cambio | NUMERIC(12,6) | Exchange rate (null for DOP invoices) |
| status | ENUM | revision, auto_aprobada, aprobada, sincronizada, error_sync, bloqueada, duplicado, escalada, declinada, eliminada |
| dgii_status | ENUM | aceptado, vigente, rechazado, no_encontrado, pendiente, no_aplica |
| gl_account | VARCHAR | GL account code (overridable per invoice) |
| cost_center | VARCHAR | Cost center (overridable per invoice) |
| erp_doc_id | VARCHAR | ERP document ID after successful sync |
| vendor_id | UUID FK | Reference to vendors table |

### invoice_items
| Column | Type | Description |
|--------|------|-------------|
| invoice_id | UUID FK | Parent invoice |
| line_number | INT | Line sequence |
| descripcion | TEXT | Item description |
| cantidad | NUMERIC | Quantity |
| precio | NUMERIC | Unit price |
| itbis_pct | NUMERIC | Tax percentage (0 or 18) |
| itbis | NUMERIC | Tax amount for this line |
| total | NUMERIC | Line total |

### vendors
| Column | Type | Description |
|--------|------|-------------|
| id | UUID | Primary key |
| company_id | UUID FK | Owning company |

| erp_card_code | VARCHAR | SAP CardCode or ERP vendor ID |
| email | VARCHAR | Vendor email |
| telefono | VARCHAR | Vendor phone |

### erp_sync_queue
| Column | Type | Description |
|--------|------|-------------|
